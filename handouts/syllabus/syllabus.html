Course Description

An exploration of fundamental data structures including stacks, queues, trees, graphs, and dictionaries will be covered using the Java programming language. An introduction to formal algorithmic complexity that compares and contrasts different algorithmic designs will also be covered.

Course Objectives

On successful completion of this course you will have better problem solving skills, be able to design and implement algorithms and analyze algorithmic complexity for solving a problem, and be comfortable programming with several data structures, including their appropriate usage. This course should prepare you to continue with more advanced computer studies and writing more substantial programs.

Prerequisites CMPT 201. You need to be familiar with Java syntax and basic programming skills. Basic algebra skills are also expected. Topics

We will cover the following topics this semester. A common theme for these topics is the separation of abstraction and implementation.

  1. Java interfaces
  2. Data structures (sets, lists, stacks, queues, trees, graphs, dictionaries)
  3. Unit testing
  4. Algorithmic complexity using Big-Oh notation

Coursework Labs

We will have one lab each week. Each lab will have one or more learning goals. Comprehension of the learning goals will determine the lab's grade. The four grade levels are detailed below.If you submit your work at least one day before the due date you will be provided feedback and be allowed to resubmit it.

E: Excellent. The work demonstrates a thorough understanding of the material. It is complete, correct, and utilizes good programming principles.

M: Meets expectations. The student comprehends both the problem and the solution. There may be minor (e.g. off-by-one) errors or incompletions.

R: Needs revisions. The student's solution is functional and demonstrates a partial grasp of the learning objectives. However, there remain significant gaps in their understanding.

N: Not assessable. The work is severely incomplete or not functional. There are too many issues to justify feedback on every one.

Homework

There will be several homework assignments, for which you will have one to three weeks to complete. Homework assignments will be appropriately weighted based upon the amount of work required. You may hand in assignments late with the penalty of 5% per day (weekends count as one day). A grading rubric will be provided on the Canvas dropbox for each homework assignment.

Late allowances will be made for serious illnesses and emergencies. If you must turn in an assignment late, make arrangements in advance or notify me as soon as possible.

Grading

Programming and lab assignments will be graded using the following criteria:

  1. Correctness & Completeness Does your solution correctly satisfy all stated requirements?
  2. Readability Do you use good class, method, and variable names and have well formatted code with proper comments and documentation?
  3. Testing and Debugging Is your code properly tested and debugged? This last step is often the difference between a A and a B grade in this class.

Ethics Presentations

Students will pair up to give a presentation on ethical issues related to computer science. As a team, you should explain the issue to the class. Then each student should present one side of the ethical conflict. You will also turn in a one-page summary of your topic.

In-class Activities

We will have a variety of in-class activities that involve working with a partner as you investigate and analyze how various data structures and algorithms work, are manipulated, and how they are evaluated. Completing these activities requires being prepared, attending class, and being an active and engaging participant.

College-Wide Learning Goals

This course satisfies the following college-wide learning goals:

  1. Critical, analytical and integrative thinking. A large focus of this class is developing critical and analytical skills for solving a variety of problems.
  2. Creative and reflective capacities. You will have to seek creative solutions to several problems you will encounter this semester. In other instances, you will be asked to reflect (in writing) on some of the solutions you have developed.
  3. Leadership, collaboration, and teamwork. You will work closely with a programming partner when pair programming in labs.
  4. Writing and other communication skills. You will complete several short writing assignments, as well as make a public presentation on a topic related to ethics and technology.

Program-Specific Learning Goals

This course satisfies the following learning goals for the computer science program:

  1. To understand the concepts and techniques of software design. This course will cover various topics in software design including class design and software abstraction.
  2. To develop effective problem solving skills. During the course of the semester you will encounter several programming challenges which you will have to apply problem solving skills to overcome.
